### Introduction

The Hospital Management System is a console-based software application developed using the C programming language. The purpose of this system is to simplify hospital administration by managing patient records, doctor information, appointments, and billing within a single. organized program. Hospitals generate a large volume of day-to-day information, and manually tracking this information is slow and error-prone. This project aims to demonstrate how fundamental C programming concepts structures, functions, arrays and file handling -- Can be applied to build a practival, real-world system

### Problem Statement
Most small and medium-sized hospitals and clinics still rely on manual, paper-based record keeping, which creates several practical problems:
*	Patient information is recorded and stored manually, making it prone to loss or damage.
*	Appointment scheduling is handled informally, leading to overlaps and confusion.
*	Generating and calculating bills manually is slow and increases the chance of human error.
*	Searching for a specific patient or doctor record among paper files is time-consuming.
*	There is no centralized way to generate quick reports for hospital administration.

### Objectives
The main objectives of this project are:
1.	To develop a console-based Hospital Management System using the C programming language.
2.	To manage patient registration, records, and search functionality.
3.	To maintain doctor information within the system.
4.	To provide a simple appointment scheduling mechanism.
5.	To automatically generate and calculate patient bills.
6.	To store all data permanently using file handling, so records persist between sessions.
7.	To reduce the manual workload involved in hospital record management.


### Scope of the project
The system will allow a hospital administrator to manage core hospital operations through a simple, menu-driven console interface. The proposed scope covers patient registration and management, doctor record management, appointment scheduling, bill generation, and basic report viewing. All data entered into the system will be saved to text files so that it remains available the next time the program is run.
The project is intended as an academic prototype rather than a production-grade hospital information system; therefore, advanced features such as multi-user access, networked databases, and graphical interfaces are kept outside the current scope, but are discussed as future improvements in Section 10.

### Proposed Features
The table below summarizes the core modules planned for the system:
Module	Description
Login	Administrator authentication before accessing the system.
Patient Management	Add, update, delete, and search patient records.
Doctor Management	Maintain a directory of doctor records.
Appointment	Book, view, and cancel patient appointments.
Billing	Generate and calculate patient bills.
Reports	View summarized hospital activity reports.
File Handling	Permanently save and retrieve all hospital data.
